## 1. 가격 설계 철학 (왜 이렇게 짜는가)

1. **랜드 앤 익스팬드(Land & Expand)** — CSPM(Posture)으로 **싸게 들어와서**, CWPP(Workload)·관제(XDR)로 **객단가를 키운다**. 무료 진단이 깔때기 입구, Enterprise가 출구.
2. **Free는 매출이 아니라 리드** — 무료 진단으로 잠재고객을 모으고 Pro로 전환시키는 미끼.
3. **Pro가 주력 상품** — 매출의 중심. "가장 인기" 배지로 시선을 모은다.
4. **Enterprise는 가치 기반(견적)** — 50~500계정·금융/공공은 정찰제가 아니라 협상. "문의"로 둔다.
5. **예측 가능한 과금 단위** — 고객이 "내 비용이 얼마일지" 쉽게 계산하게.

## 2. 과금 단위 결정 (핵심)

| 모듈 | 과금 단위 | 이유 |
|---|---|---|
| **Posture (CSPM)** | **클라우드 계정 수 / 월** | 고객 규모와 비례, 계산 쉬움, 멀티계정 확장 시 자연 증액 |
| **Workload (CWPP)** | **보호 워크로드(인스턴스/vCPU) 수 / 월** | 에이전트 기반, 실제 보호 대상에 비례 |
| **관제 (XDR)** | **매니지드 — 맞춤 견적** | 인력·SLA 기반 서비스라 정찰제 부적합 |

> 보조 원칙: 계정 수는 **구간(band)**으로 묶어 단가 할인(1~5 / 6~20 / 21~50 / 51+). 많이 쓸수록 계정당 단가↓.

## 4. 티어별 상세

### ▎Free — 무료 ISMS-P 진단 (리드 엔진)
- **타겟**: "우리 점수 궁금한" 모든 잠재고객.
- **포함**: AWS 1계정, 186개 점검 1회, 점수 + 요약 리포트.
- **의도된 제한**: 스케줄·생명주기·뮤트·다계정 차단 → **Pro의 가치를 체감**시켜 전환.
- **성공 지표**: 진단 신청 수 → Pro 전환율(MQL→SQL).

### ▎Starter — CSPM 입문 (전환 다리)
- **타겟**: 단일~소수 계정, 인증을 막 시작한 소규모 조직.
- **포함**: 전체 점검 + 주1회 스케줄 + 기본 생명주기 + 메일 리포트.
- **역할**: Free와 Pro 사이 진입장벽 완화. *주력은 아님* — Pro로 끌어올리는 계단.
- **가격 논리(가설)**: 계정당 월 정액. "수동 점검 인건비보다 싸다"가 기준점.

### ▎Pro — CSPM 본격 (★ 주력 상품, 매출 중심)
- **타겟**: ISMS-P 의무대상 중견기업, 보안팀 1~5명, 다계정.
- **포함**: 무제한 스케줄, **전체 생명주기(재발 추적)**, 뮤트 규칙, 점수 추세, 다계정 대시보드, Slack 알림, 경영진+기술 리포트, 우선 지원.
- **CWPP add-on 후크**: 여기서 "Workload 모듈 추가하면 런타임까지 막힙니다" 업셀 시작.
- **가격 논리(가설)**: 계정 구간별 단가, 연간 약정 할인. "심사 컨설팅 1회 비용 ≈ Pro 수개월"로 가치 정당화.

### ▎Enterprise — CNAPP 통합 (★ 객단가 핵심)
- **타겟**: 50~500계정, 금융·공공, 멀티클라우드.
- **포함**: Pro 전체 + **CWPP(Host IPS) 기본 포함** + 관제(XDR) 옵션 + SSO/RBAC·감사로그·전담 CSM·SLA·온프렘/프라이빗 옵션·맞춤 리포트.
- **삼중 방패 완성**: Posture + Workload + 관제 = 하나의 요새.
- **가격 논리**: **가치 기반 견적**(정찰제 X). 계정 수·워크로드 수·관제 범위·SLA로 산정, 협상 여지 확보.

## 5. CWPP 업셀 / Land & Expand 경로 (매출 성장 엔진)

```
무료 진단(Posture)  →  Starter/Pro(Posture 유료)  →  + Workload(CWPP) add-on  →  Enterprise(CNAPP + 관제)
   리드 확보            CSPM 정착·신뢰              ARPU 확대(런타임 방어)        최대 객단가(삼중 방패)
```

- **핵심 메시지(업셀)**: *"설정 점검(Posture)으로 '잘못된 문'은 찾았습니다. 그런데 공격은 **실행 중인 워크로드**에서 일어납니다. Workload 모듈로 런타임까지 막으세요."*
- **트리거 시점**: Pro 고객이 ① 점수 안정화 후 ② 사고/모의해킹 경험 ③ 인증 갱신기 → CWPP 제안 최적기.
- **번들 할인**: Posture + Workload 동시 계약 시 통합 할인 → Enterprise로 자연 승급.

## 7. 가격 공개 전략 (권고)

- **권고**: **기능 매트릭스는 공개 + 가격은 "문의/견적"** (단, Free·Starter는 진입장벽 낮추려 "₩X부터" 앵커 표기 검토).
- **이유**: 한국 B2B 보안은 견적 문화가 강하고, 가격 노출 시 경쟁사가 즉시 베껴 하단 침투. 동시에 **완전 비공개는 불신** → "기능은 투명, 가격은 상담" 절충이 전환율·신뢰의 균형점.
- **Pro에 "가장 인기" 배지** → 시선 유도(앵커링).
- **연간 약정 할인**(예: 2개월분) → 캐시플로·리텐션 개선.

## 8. 내부 가격 책정 3축 (확정 전 반드시 산정)

1. **원가 바닥(floor)**: 계정당 점검에 드는 인프라·API·운영 비용 → 이보다 낮으면 손해.
2. **경쟁 천장(ceiling)**: 글로벌 CSPM 대비 **체감상 더 저렴**해야 함(우리의 가격 무기).
3. **가치 기준(value)**: 고객이 절약하는 인건비·컨설팅비·인증 실패 리스크 → "ROI로 정당화 가능한 상한".
   → 셋을 겹쳐 **floor < 우리 가격 < min(ceiling, value)** 구간에서 결정.
