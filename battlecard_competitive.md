# gnFortress 경쟁 배틀카드 (Sales Battlecard)
### by 가디언넷(Guardiannet) · 영업·세일즈 내부 전용

> 용도: 고객이 "그거 ○○랑 뭐가 달라요?"라고 물을 때 **즉답**하기 위한 영업 무기.
> 사용법: 미팅 전 해당 경쟁자 카드 1장만 훑고 들어갈 것. **경쟁사 비방 금지** — 사실 기반 차별화만.

---

## 0. 공통 — gnFortress "3대 무기" (모든 경쟁 상황 공통 메시지)

1. **"ISMS-P를 아는 유일한 CNAPP"** — 글로벌 툴은 한국 인증 기준을 모릅니다. 우리는 ISMS-P 2023 항목별 매핑.
2. **"찾는 데서 끝나지 않고, 조치완료·재발까지 추적"** — 발견사항 생명주기 관리 = 심사 증빙이 완벽.
3. **"국내 금융권 1위가 만든 검증된 엔진"** — 4,000+ 워크로드를 지켜온 가디언넷의 실전 신뢰.

> 한 문장 엘리베이터 피치:
> *"글로벌 CNAPP만큼 똑똑하지만, 한국 인증(ISMS-P)을 알고, 한국어로 심사를 대응하며, 합리적 가격에 금융권에서 검증된 솔루션입니다."*

---

## 1. vs **Wiz / Prisma Cloud (Palo Alto)** — 글로벌 CNAPP

**한 줄**: 기술력·브랜드 강한 글로벌 강자. 단, 비싸고 한국 인증·심사·지원이 약함.

| 그들의 강점 (인정할 것) | 우리가 이기는 지점 |
|---|---|
| 에이전트리스 스캔, 그래프 분석, 글로벌 브랜드 | **ISMS-P 2023 항목별 매핑 리포트** (그들은 한국 인증 기준 자체가 없음) |
| 방대한 기능 폭, 글로벌 엔터프라이즈 레퍼런스 | **한국어 심사 대응 리포트** + 국내 즉시 지원 체계 |
| 멀티클라우드 커버리지 | **합리적 가격** — 그들은 중견기업엔 고가·오버스펙 |
| | **데이터 주권** — 해외 SaaS 대비 국내 처리 안심 |

**🪤 심어둘 함정 질문 (고객이 Wiz/Prisma에게 던지게 만들 것):**
- "그 솔루션이 **ISMS-P 2023 인증기준에 항목별로 매핑된 리포트**를 한국어로 줍니까?"
- "우리 **심사원이 인정하는 증빙 형식**으로 나옵니까?"
- "발견된 취약점을 **고친 뒤 재발했는지까지** 추적·증빙해 줍니까?"
- "연간 비용이 우리 클라우드 규모에 **과하지 않습니까?** (좌석·계정 과금 구조 확인)"

**반론 대응:**
- *"Wiz가 더 유명하잖아요"* → "맞습니다. 글로벌 대기업엔 좋은 선택이죠. 다만 고객님의 **핵심 목표가 ISMS-P 인증**이라면, 인증 기준을 모르는 도구에 비싼 비용을 쓰는 것보다, **인증에 맞춰 설계된 솔루션 + 무료 진단**으로 먼저 점수를 확인해 보시는 게 합리적입니다."

## 2. vs **국내 대형 (SK쉴더스 / 안랩 / 이글루 등)** — 관제(MSSP) 중심

**한 줄**: 브랜드·관제 인력 강함. 단, 관제 서비스 중심이라 비싸고 셀프서비스 SaaS가 아님.

| 그들의 강점 (인정할 것) | 우리가 이기는 지점 |
|---|---|
| 강력한 브랜드, 풍부한 레퍼런스, 관제 인력 | **셀프서비스 SaaS = 빠른 도입·합리적 비용** (대형 SI·관제 계약은 무겁고 비쌈) |
| 종합 보안(물리·네트워크 등) | **클라우드 네이티브 전문** — CSPM/CWPP/CNAPP에 집중 |
| 인지도 | **가디언넷도 금융권 1위** — 신뢰는 대등, 민첩성은 우위 |

**🪤 함정 질문:**
- "그 관제 서비스에 **연간 얼마**가 들고, 도입까지 **몇 개월** 걸립니까?"
- "**우리가 직접** 점수·현황을 실시간으로 보는 대시보드가 있습니까, 아니면 그쪽 리포트를 기다려야 합니까?"
- "ISMS-P **항목별 자동 점검·생명주기 추적**이 제품으로 제공됩니까, 사람이 수동으로 합니까?"

**반론 대응:**
- *"대형사가 더 안심돼요"* → "종합 관제가 필요하시면 좋은 선택입니다. 다만 **클라우드 ISMS-P 점검**만 놓고 보면, 무거운 관제 계약 없이 **오늘 바로 셀프서비스로 시작**하고, 필요할 때 방어(Workload)·관제(XDR)로 확장하는 게 비용·속도 면에서 유리합니다. 우리도 금융권에서 검증됐습니다."

---

## 3. vs **수동 점검 / 컨설팅 (현상유지)** — *가장 흔한 진짜 경쟁자*

**한 줄**: "지금 사람이 하고 있다 / 컨설팅에 맡긴다" — 익숙함이 무기. 하지만 구조적 한계 명확.

| 그들의 "강점" | 우리가 이기는 지점 |
|---|---|
| 익숙함, 당장 추가 비용 없어 보임 | **매일 자동 점검** vs 분기 1회 → 시점 공백 제거 |
| 컨설턴트가 봐주니 안심 | **누락·휴먼에러 제거** (186개 항목 기계 점검) |
| | **생명주기 추적**으로 재발·조치완료 자동 증빙 (수작업 증빙 불필요) |
| | 실제론 **인건비·야근 비용이 더 큼** — 숨은 비용 가시화 |

**🪤 함정 질문:**
- "마지막 점검이 **언제**였고, 그 사이 바뀐 설정은 누가 잡습니까?"
- "심사 때 '이 취약점 고쳤다'를 **무엇으로 증빙**하십니까? 수기 정리에 며칠 쓰십니까?"
- "한 번 고친 게 **다시 열렸는지** 누가 추적합니까?"

**반론 대응:**
- *"지금도 어떻게든 하고 있어요"* → "그 노력 충분히 압니다. 그래서 **부담 없이 무료 진단**으로 *지금 점수*만 먼저 확인해 보시죠. 수동으로 놓치고 있던 항목이 몇 개 나오는지 보시면, 자동화 가치가 숫자로 보일 겁니다."

> 💡 이 경쟁자에겐 **무료 진단이 최강의 무기**. "비교"가 아니라 "현실 직시"를 시킴.

## 4. vs **AWS 네이티브 (Security Hub / Config / Audit Manager)**

**한 줄**: AWS 통합·저렴. 단, ISMS-P를 모르고 한국 심사 대응이 안 됨.

| 그들의 강점 (인정할 것) | 우리가 이기는 지점 |
|---|---|
| AWS 기본 통합, 저비용, CIS/PCI 기본 제공 | **ISMS-P 2023 특화 매핑** (AWS 기본은 한국 인증 기준 없음) |
| | **한국어 심사용 리포트 + 경영진 1페이지 점수** |
| | **생명주기 추적·뮤트·멀티계정 집계 UX** — 네이티브는 운영이 복잡하고 분산됨 |

**🪤 함정 질문:**
- "Security Hub 결과가 **ISMS-P 인증기준에 매핑**돼서 나옵니까?"
- "심사원에게 줄 **한국어 증빙 리포트**가 바로 나옵니까?"
- "수십 개 계정 결과를 **하나의 점수·하나의 화면**으로 집계해 줍니까?"

**반론 대응:**
- *"AWS 기본 도구로 충분한데요"* → "기초 점검엔 좋습니다. 다만 그건 **'보안 일반'**이지 **'ISMS-P 인증'**이 아닙니다. 인증 항목 매핑·한국어 증빙·재발 추적은 직접 만드셔야 하는데, 그 운영 비용이 우리 솔루션보다 큽니다."

---

## 5. 절대 하지 말 것 (Don'ts)

- ❌ 경쟁사 **비방·허위 폄하** — 신뢰만 잃음. 항상 "그들 강점 인정 → 우리 차별점"
- ❌ **기능 나열 경쟁** — Wiz와 기능 수로 싸우지 말 것. **"ISMS-P 인증 목표"**라는 프레임으로 끌어올 것
- ❌ 확인 안 된 **가격·수치 단정** — "통상 고가" 정도로, 구체 숫자는 고객 환경 확인 후
- ❌ **"인증 보장"** 류 표현 — 우리는 점검·증빙을 돕는 것이지 합격을 보장하지 않음

## 6. 모든 미팅 마무리 한 마디
> *"비교는 나중에 하셔도 됩니다. 부담 없이 **무료 ISMS-P 진단**으로 지금 우리 클라우드 점수부터 확인해 보시죠. 그 리포트 한 장이면 어느 솔루션이 맞는지 스스로 보일 겁니다."*
