# gnFortress 셀프서비스 온보딩 플로우 설계서
### 가입 → AWS 연결(읽기전용) → 첫 점검 → 리포트

> 용도: 셀프서비스 전환의 핵심 — 고객이 스스로 가입하고 안전하게 자기 AWS를 연결해 점검까지 도달하는 흐름.
> 연계: 한도는 `entitlements_design.md`(B), 화면은 랜딩(A).
> 3대 목표: ① **마찰 최소화**(이탈 방지) ② **소유권 검증**(남의 계정 스캔 차단) ③ **보안 신뢰**(읽기전용).

---

## 1. 전체 플로우 (한눈에)

```
[S1] 가입            업무 이메일 + 비밀번호 → 이메일 인증
   │
[S2] 조직 생성        회사명·담당자 → 워크스페이스 생성 (플랜=Starter 무료 기본)
   │
[S3] AWS 연결         "AWS 연결" → CloudFormation 원클릭 → 내 계정에 읽기전용 역할 생성
   │
[S4] 소유권 검증      gnFortress가 STS AssumeRole(ExternalId) 성공 → 연결 확인 ✓
   │
[S5] 첫 점검 실행     "지금 점검" 버튼 → ISMS-P 186개 항목 스캔 (진행률 표시)
   │
[S6] 리포트 확인      준수 점수·취약 목록·조치 가이드 (화면 + PDF)
   │
[S7] 업그레이드 동선  한도 도달/추가 계정 시 → Pro 업그레이드 or Enterprise 상담
```

---

## 2. 단계별 상세

### S1. 가입 (목표: 30초)
- **입력 최소화**: 업무 이메일 + 비밀번호(또는 Google/Microsoft SSO).
- **업무 이메일 인증**: 무료메일(gmail/naver) 가입 허용 여부 정책화 — 남용·가짜 리드 방지 위해 **업무 도메인 권장**, 인증 메일 클릭 필수.
- 신용카드 요구 ❌ (Starter 무료).

### S2. 조직(워크스페이스) 생성
- 회사명·담당자명·연락처 → 워크스페이스 1개 생성, 기본 플랜 **Starter**.
- 생성자 = 첫 admin seat.

### S3. AWS 연결 (가장 중요한 마찰 지점)
- **방식: CloudFormation 원클릭** (수동 IAM 생성 금지 — 이탈 1순위).
  - 화면의 "AWS에 연결" 버튼 → 고객 AWS 콘솔의 CloudFormation **빠른 생성 링크**로 이동(템플릿 URL + 파라미터 프리필).
  - 스택이 **읽기전용 IAM 역할** 생성:
    - **신뢰 정책(Trust)**: gnFortress의 AWS 계정만 `sts:AssumeRole` 허용 + **ExternalId**(워크스페이스별 고유값)로 confused-deputy 방지.
    - **권한**: AWS 관리형 읽기 정책(예: `SecurityAudit`, `ViewOnlyAccess`) 범위 내 — ISMS-P 점검에 필요한 **읽기 권한만**. 쓰기·삭제 권한 없음.
  - 스택 출력의 **Role ARN**을 화면에 붙여넣기(또는 자동 콜백).
- **대안 연결 방식(선택)**: Terraform 모듈 제공(IaC 쓰는 고객), 수동 가이드(폴백).

### S4. 소유권 검증 (보안·법적 안전장치)
- gnFortress 백엔드가 입력된 Role ARN으로 **STS AssumeRole(ExternalId 포함)** 시도 → **성공해야만 연결 완료**.
- **소유권 증명 원리**: 고객이 *자기 계정에* 역할을 만들고 ExternalId를 심을 수 있다는 것 = 그 계정의 통제권 보유 증명. → **남의 계정 ID만으로는 스캔 불가**(무단 정찰 차단).
- 연결 성공 시 "계정 ✓ 연결됨" 상태. 실패 시 원인별 안내(S6 엣지케이스).

### S5. 첫 점검 실행
- 빈 상태(empty state) 화면에 **"지금 점검하기" 단일 CTA** — 고민 없이 누르게.
- 실행 중 **진행률·예상 시간** 표시(체감 대기 감소). 비동기 작업.
- 한도 적용(B): Starter는 월 5회 온디맨드 — 잔여 횟수 표시.

### S6. 리포트 확인 (Aha-moment)
- 완료 즉시 **준수 점수(0~100) + 취약 목록 + 우선순위 조치 가이드**.
- **PDF 내보내기**(경영진 보고용). 화면 상단에 **사용량 게이지**(계정 1/1, 온디맨드 4/5 등).
- 다음 행동 유도: "스케줄 설정" / "다른 계정 추가(→한도 시 업셀)".

### S7. 업그레이드 동선
- 계정 추가·HostIPS·고급기능 시도 시 한도 모달 → **Pro 셀프 업그레이드** 또는 **Enterprise 상담**(B의 §4·§5).

## 3. 보안 설계 요약 (가입 화면·문서에 명시)

| 항목 | 설계 |
|---|---|
| 권한 범위 | **읽기 전용**(AssumeRole + read-only 정책). 설정 변경·삭제 불가 |
| confused deputy 방지 | **ExternalId**(워크스페이스 고유) 필수 |
| 최소 권한 | ISMS-P 점검에 필요한 읽기 권한만. 과도 권한 요구 금지 |
| 소유권 검증 | 고객이 직접 자기 계정에 역할 배포해야만 연결 → 무단 스캔 차단 |
| 자격증명 보관 | 장기 키 저장 금지, **STS 임시 자격증명**만 사용 |
| 데이터 위치 | 국내 처리(데이터 주권 메시지) |
| 남용 방지 | 온디맨드 rate limit + 업무메일 인증 |
| 해지 | 고객이 CloudFormation 스택 삭제 = 역할 제거 → 즉시 접근 차단 |

## 4. 마찰 제거 장치 (전환율 핵심)

- **진행 표시기**: 가입→연결→점검 3/4단계 진행바 상시 노출(완주 욕구).
- **원클릭 IaC**: 수동 IAM 절대 금지. CloudFormation 빠른생성 + 파라미터 프리필.
- **빈 상태 디자인**: "아직 점검 전이에요 — 지금 점검하기" 단일 행동.
- **즉시 가치**: 첫 리포트까지 도달 시간(TTV)을 최우선 지표로. 목표 **가입 후 10분 내 첫 점수**.
- **가이드/툴팁**: 각 단계에 30초짜리 안내·스크린샷.

## 5. 엣지케이스 / 실패 처리

| 상황 | 처리 |
|---|---|
| Role ARN 오입력 / AssumeRole 실패 | "역할을 찾을 수 없습니다 — ExternalId·신뢰계정 확인" + 재배포 가이드 |
| 권한 부족(일부 API 거부) | 점검은 진행하되 해당 항목 **ERROR(확인불가=결함 보수처리)** 표시 |
| 리전 미선택/불일치 | 점검 대상 리전 선택 UI, 기본 리전 안내 |
| 스택 생성 권한 없는 IAM 사용자 | "AWS 관리자 권한으로 배포 필요" 안내 |
| 중복 계정 연결 | 동일 계정 재연결 방지·기존 연결 안내 |
| 무료 한도 소진 | 다음달 초기화 안내 + 업그레이드 CTA |

## 6. 기술 구성요소 체크리스트 (제품·엔지니어링)

- [ ] **인증/계정**: 회원가입·로그인·이메일 인증·SSO(선택), 워크스페이스·seat·RBAC.
- [ ] **AWS 연결**: CloudFormation 템플릿(읽기전용 Role + ExternalId), Quick-create 링크 생성, Role ARN 입력/콜백.
- [ ] **STS AssumeRole**: ExternalId 검증, 임시 자격증명으로 점검 수행.
- [ ] **점검 실행기**: 비동기 작업·진행률·온디맨드 rate limit(B의 D4).
- [ ] **엔타이틀먼트 게이트**: plan→limits 매핑 enforcement(B의 §7).
- [ ] **리포트**: 점수·취약목록·PDF 내보내기·사용량 게이지.
- [ ] **결제**: 카드 + 세금계산서/계좌이체(Pro), 업그레이드/다운그레이드 처리.
- [ ] **알림**: 가입·연결·점검완료 메일, Slack(Pro+).
